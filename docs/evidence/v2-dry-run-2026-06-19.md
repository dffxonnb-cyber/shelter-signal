# Shelter Signal V2 Dry-run Evidence · 2026-06-19

## Status

Shelter Signal V2 is documented as a **local preview and dry-run track** for alert-candidate and digest generation.

This evidence does not claim a Production notification service.

It records a local dry-run validation path for:

```text
Docker PostgreSQL
→ SQL models
→ mart.alert_candidates
→ daily digest JSON/HTML preview
→ optional local n8n or Mailpit validation
```

## Production / V2 Boundary

Current Production remains the live-first public-data service:

```text
Vercel /api/notices
→ data.go.kr rescued-animal notice API
→ KST freshness normalization
→ current / urgent / protected / archive
→ region / page filtering
→ live/cache/fallback metadata
```

V2 is separate from the Production notice pipeline.

V2 does not change the live-first `/api/notices` architecture.

## What This Evidence Proves

```text
alert-candidate selection logic can be previewed locally
digest output shape can be reviewed before real delivery
JSON/HTML preview can be generated without real recipients
V2 can be validated separately from Production live-first notices
```

## What This Evidence Does Not Prove

```text
real external email delivery
SMS or push notification delivery
real recipients or subscription management
Production notification schedule
Production monitoring or delivery SLA
```

## Safety Boundary

This evidence must not include:

```text
SMTP credentials
Gmail OAuth tokens
real recipient emails
service keys
database URLs
.env values
generated private exports
```

Local dry-run output is for reviewer interpretation only.

Do not present local Mailpit, n8n dry-run, or digest preview as Production notification readiness.

## Reviewer Interpretation

Shelter Signal V2 should be read as a local proof-of-flow:

```text
candidate signal
→ digest preview
→ local validation
→ documented delivery boundary
```

It is not an operated alerting product.

The value of this V2 track is showing that the alert logic can be inspected safely before any real notification, account, subscription, or delivery system exists.
